# NapCat 每小时重启 + 登录态失效模式

## 现象

NapCat QQ Bot ({{BOT_QQ_ID}}) 每整点自动重启一次。重启后快速登录失败，因为缓存的登录 token 已过期。

## 日志关键行

### 重启触发
```
05-14 10:00:07 [info] 正在快速登录  {{BOT_QQ_ID}}
```
前面跟着 GPU 进程退出 + `NapCat Shell App Loading...` — 说明 NapCat 进程被重启了。

### 重启后快速登录失败
```
[error] 快速登录错误： 登录态已失效，请重新登录。
```
或：
```
[error] 快速登录错误： 你的用户身份已失效，为保证账号安全，请你重新登录。
```

### 首次启动（无历史记录）
```
[error] 快速登录失败，未找到该 QQ 历史登录记录，将尝试密码回退登录
```
只有在全新安装或无 token 缓存时出现。

### 在线久了被踢下线
```
[error] bot | [KickedOffLine] [下线通知] 你的帐号当前登录已失效，请重新登录。
```

## 时间线（从日志提取）

| 时间 | 事件 |
|------|------|
| 05-11 03:58 | 首次启动，无历史记录，二维码扫码登录成功 |
| 05-11 04:00 ~ 05:00 | 连续 4 次快速重启，每次间隔 ~2min，均因 token 失效失败 |
| 05-11 04:31 ~ 04:51 | 继续尝试快速登录（仍在 2min 间隔重试） |
| 05-11 08:40 | NapCat 再次启动 → 快速登录失败 → 进入 2min 循环 |
| 05-11 08:40 ~ 15:53 | **持续 7 小时重试**（每 ~2min 一次 Login Error），直到被踢下线后才触发新的重启 |
| 05-13 10:00 | 首次出现**每小时整点重启**模式 |
| 05-13 10:00 ~ 14:00 | 每个整点重启 → 快速登录失败 → 2min 循环 |
| 05-13 15:53 | [KickedOffLine] |
| 05-13 16:00 ~ 22:00 | 继续每小时整点重启 + 快速登录失败循环 |
| 05-14 00:00 ~ 10:00 | 持续整点重启 → 快速登录失败 |

## 恢复方式

用户通过以下方式之一手动重新登录：
1. **WebUI** (port 6099) — 扫码/密码登录
2. **二维码** — 重新启动 NapCat 生成二维码 → 手机 QQ 扫码
3. **密码环境变量** — `NAPCAT_QUICK_PASSWORD` env var（有 SMS 验证风险）

用户重新登录后，快速登录认证恢复，直到下一轮每小时重启时 token 再次过期。

## 看门狗检测

`qq_watchdog.py` 每分钟运行一次，从日志文件末尾扫描新增行，检测两类告警：
- `快速登录失败` / `快速登录错误` — 重启后登录失败
- `KickedOffLine` / `被踢下线` — 在线被踢

## 已知

- 每小时重启的来源不确定（可能是 systemd timer 或外部 cron），用户确认"这是确定的每小时重启"
- 看门狗只进行检测和通知，不自动修复
